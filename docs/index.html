CivicCode

Municipal code and ordinance access, designed to answer code questions with exact citations while keeping legal interpretation with city staff.

Durable popular-questions discovery runtime online on the Docker/PostgreSQL path, alongside related-section navigation aids, mock-city codifier contracts, staff code lifecycle workspace, staff source registry, records-ready export and accessibility hardening, local import, public lookup, CivicClerk handoff, plain-language summaries, staff workbench foundation, citation-grounded Q&A, citation contract, search and permalink, section/version, database-backed source registry, runtime foundation, and canonical schema foundations - legal determinations, live LLM calls, bundled vendor credentials, CivicAccess runtime dependency, and automatic ordinance codification not available

What it will do

  • Import authoritative municipal code sources from local fixtures and file drops.
  • Preserve section versions and effective dates.
  • Export records-ready section packages with citations and source provenance.
  • Answer routine code questions with citations.
  • Label summaries as non-authoritative.

What it will not do

  • Replace the official codifier.
  • Give legal advice.
  • Automatically codify ordinances.
  • Start CivicZone runtime work.

What ships today

  • Installable Python package.
  • FastAPI runtime foundation with / and /health.
  • Published civiccore v0.22.0 release-wheel dependency.
  • Staff-header-protected source registry mutations and staff source reads.
  • Staff source registry workspace at /staff/sources with empty, active, stale, failed, and access-required states.
  • Staff code lifecycle workspace at /staff/code with access-required, empty, readiness-snapshot, section-card, draft-summary, and pending-codification states.
  • Canonical SQLAlchemy metadata and Alembic migrations for ten civiccode.* tables.
  • Official source registry API for codifier/source metadata, provenance, source-state checks, and optional persistence through CIVICCODE_SOURCE_REGISTRY_DB_URL.
  • Section/version lifecycle APIs for adopted, historical, and pending text boundaries.
  • Public-safe search and stable section permalink APIs.
  • Staff-approved popular questions that link only to cited adopted code.
  • Durable popular-question storage through CIVICCODE_SOURCE_REGISTRY_DB_URL for Docker/PostgreSQL demos.
  • Related materials from explicit public cross-references, labeled as navigation aids and not legal determinations.
  • Deterministic citation/refusal objects with section, version, source, and effective-date fields.
  • Citation-grounded Q&A harness for questions that resolve to one adopted section and active source.
  • Staff interpretation notes, staff Q&A context, and staff workbench audit events.
  • Staff-approved plain-language summaries labeled non-authoritative with non_authoritative_explanation.
  • Authoritative code text remains visible beside approved summaries.
  • CivicClerk handoff intake with pending codification warnings and likely conflict signals.
  • Resident-facing public code lookup surface under /civiccode for the "Read code" workflow.
  • Search, empty, legal-advice refusal, stale-source warning, and section detail states.
  • Staff-only local import jobs for CSV/file-drop bundles and official HTML extract fixtures.
  • Import retry, visible failure records, imported-tree verification, and provenance report endpoints.
  • No outbound network dependency, Redis/Celery worker, or vendor credentials required for local import.
  • Staff-controlled codifier sync readiness endpoints with schedule validation, SSRF-safe host checks, and supported connector validation.
  • Delta request planning for Municode, American Legal Publishing, Code Publishing Company, and General Code.
  • Local codifier payload runs through the import path with CivicCore circuit-breaker health, shared source-list health projection, and actionable operator copy.
  • Reusable mock-city codifier contracts for Municode, American Legal Publishing, Code Publishing Company, and General Code.
  • Mock-city report reuses CivicCore municipal IdP and backup-retention contracts and writes a secret-free JSON report.
  • Records-ready export API and accessible HTML export page for adopted sections.
  • Docker Compose product path with PostgreSQL 17 plus pgvector, migration startup, and City of Brookfield seeded demo data enabled by CIVICCODE_DEMO_SEED=1.
  • Docker/PostgreSQL backup-restore rehearsal with pg_dump, pg_restore, restored-table verification, civiccode-docker-backup-manifest.json, and Windows/Bash launchers.
  • Semantic headings, labels, focus styling, and print-friendly export output.
  • CivicAccess integration notes without a shipped CivicAccess runtime dependency.
  • Pending ordinance language is not adopted law and does not replace codified text.
  • Staff context is marked staff_only_do_not_publish; public answers stay citation_grounded.
  • No legal advice, live LLM calls, or automatic ordinance codification yet. Staff notes remain staff-only.
