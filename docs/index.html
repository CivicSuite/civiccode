CivicCode

Municipal code and ordinance access, designed to answer code questions with exact citations while keeping legal interpretation with city staff.

Local import and connector hardening foundation online on top of the public lookup, CivicClerk handoff, plain-language summaries, staff workbench foundation, citation-grounded Q&A, citation contract, search and permalink, section/version, source registry, runtime foundation, and canonical schema foundations - legal determinations, live LLM calls, live codifier sync, and automatic ordinance codification not available

What it will do

  • Import authoritative municipal code sources from local fixtures and file drops.
  • Preserve section versions and effective dates.
  • Answer routine code questions with citations.
  • Label summaries as non-authoritative.

What it will not do

  • Replace the official codifier.
  • Give legal advice.
  • Automatically codify ordinances.
  • Start CivicZone runtime work.

What ships today

  • Installable Python package.
  • FastAPI runtime foundation with / and /health.
  • Exact civiccore==0.2.0 dependency pin.
  • Canonical SQLAlchemy metadata and Alembic migrations for ten civiccode.* tables.
  • Official source registry API for codifier/source metadata, provenance, and source-state checks.
  • Section/version lifecycle APIs for adopted, historical, and pending text boundaries.
  • Public-safe search and stable section permalink APIs.
  • Deterministic citation/refusal objects with section, version, source, and effective-date fields.
  • Citation-grounded Q&A harness for questions that resolve to one adopted section and active source.
  • Staff interpretation notes, staff Q&A context, and staff workbench audit events.
  • Staff-approved plain-language summaries labeled non-authoritative with non_authoritative_explanation.
  • Authoritative code text remains visible beside approved summaries.
  • CivicClerk handoff intake with pending codification warnings and likely conflict signals.
  • Resident-facing public code lookup surface under /civiccode for the "Read code" workflow.
  • Search, empty, legal-advice refusal, stale-source warning, and section detail states.
  • Staff-only local import jobs for CSV/file-drop bundles and official HTML extract fixtures.
  • Import retry, visible failure records, imported-tree verification, and provenance report endpoints.
  • No outbound network dependency, Redis/Celery worker, or live codifier sync required for local import.
  • Pending ordinance language is not adopted law and does not replace codified text.
  • Staff context is marked staff_only_do_not_publish; public answers stay citation_grounded.
  • No legal advice, live LLM calls, or automatic ordinance codification yet. Staff notes remain staff-only.
